# 视频生成质量护栏

## 目标

网页系统自动生成数字人视频时，必须把“底片生成”和“字幕包装”分开处理：

- 底片只生成人物、背景、动作和口播状态，不允许模型把脚本文字画进视频。
- 字幕由系统后期统一烧录，使用中文字体和参考口播字幕风格。
- 成片必须先抽帧检查，不能只看任务成功状态。

## 火山即梦底片规则

`volcengine-jimeng-digital-human` 默认使用干净底片策略：

```text
allow_script_prompt=false
prompt_policy=clean_no_text_plate
```

代码会避免把完整脚本或口播稿传给火山 prompt，并自动追加禁止画面文字的约束：

```text
不要出现任何文字、字幕、标题、关键词、数字、Logo、贴纸、UI、白色大字、说明文字或屏幕文字。
后期字幕由系统单独添加。
```

原因：火山小云雀有参考生成会根据 prompt 或参考视频生成画面内大字，后期无法自然移除。

## 字幕规则

数字人口播默认字幕样式：

```text
template=digital_human
label=参考口播字幕
font=Noto Serif CJK SC
style=底部单行短句、白字、黑描边、金色关键词
```

禁止用大黑块遮挡模型生成的文字。黑块只能作为临时排障，不允许作为正式字幕设计。

## 质检要求

每次自动生成后至少检查：

1. 底片是否有模型自带大字、标题、贴纸、字幕或 UI。
2. 中文字幕是否显示正常，不能出现方块。
3. 字幕是否接近参考视频风格，不能使用粗暴黑块。
4. 字幕是否遮挡脸、嘴、手势或核心画面。
5. 若底片已有内嵌文字，必须重新生成干净底片，不能靠后期遮丑。

## 事故记录

2026-06-28 测试发现的问题：

- 火山底片把脚本内容生成成画面内大字。
- 后期字幕因字体为 Arial，中文显示为方块。
- 黑色字幕块虽然能遮挡问题，但观感差，不符合参考视频风格。

已修复：

- 火山 prompt 默认改为干净底片策略。
- 数字人字幕模板改为 `Noto Serif CJK SC`。
- 字幕模板改为参考口播短句样式。
