# 何依依参考视频工程级逐帧解析

## 结论

原视频可以作为固定时间轴进行高保真人物替换，但不能再使用“整条 Seedance 重生成”的方案。整条重生成会同时改变背景、字幕、B-roll、剪辑节奏和音效，无法达到除人物外一致。

最接近目标的生产链路是：原片作为母版 -> 只对人物可见区间做动作驱动 -> 原 B-roll 和结尾原帧回填 -> 原标题、字幕、关键词卡片回填 -> 原音轨原时长复用。

目标人物必须使用系统里最新的火山企业数字资产：数字人 `#6`、Asset Group `group-20260711223057-gddps`、`asset://asset-20260711223536-zgjjn`，当前状态 `active`。旧素材 `#10`、`#25` 和 `#31` 不得再作为人物生成输入；其中 #25/#31 正是人物过大、偏右和手臂截断的旧版本。

最新企业资产只负责黄丽身份。原视频按人物可见区间切成 5–10 秒动作参考片，连同企业资产一起提交给 Seedance；背景、字幕、插片和卡片仍由确定性后期合成恢复。

## 图层模型

1. L0：原固定机位背景或基于原背景重建的 clean plate。
2. L1：最新黄丽企业资产驱动的人物层，身份固定为 `asset://asset-20260711223536-zgjjn`，动作、表情和镜头节奏参考原人物。
3. L2：原视频顶部插画、后台截图、关键词卡片和全屏 B-roll。
4. L3：原视频标题、逐句字幕、重点词高亮和结尾 CTA。
5. L4：原音轨，保持 70.635 秒不重编码或仅最终封装一次。

直接在原片上做“全帧换人”不可行，因为 L2/L3 已烘焙在像素里；直接使用生成模型也会重画 L0。必须按时间段选择替换人物、回填叠层或完整保留原帧。

## 人物与机位约束

- 原人物可见主体大致范围：`x=55..530, y=365..1024`。
- 原头部大致范围：`x=202..391, y=363..620`。
- 使用 macOS Vision 每 0.5 秒检测后，人脸框中位数为 `x=226.11, y=423.17, w=143.10, h=143.10`；横向转头范围约 `-0.131..0.114 rad`。
- 最新黄丽资产生成画面的人脸中心、肩宽、坐姿高度和镜头透视必须与上述基线对齐，中心误差不超过 8px，脸部尺度误差不超过 5%。
- 镜头固定、平视、无焦距变化；动作集中在表情、头部小幅转动和胸前手势。
- 黄丽人物使用最新企业资产，原样保持资产中的肩长侧分发型、绿色上衣、饰品、年龄感和自然肤质；不得再用提示词强行改成旧版深蓝 Polo，也不得复用何依依的发型、服装或身体轮廓。
- 动作驱动应分人物连续区间提交；全屏 B-roll 和 CTA 不应进入动作模型，避免额外成本和画面漂移。

## 字幕与图形

- 持续标题：`旺季心法 / 一招制胜`，两行衬线粗体，暖白色，深色硬阴影，位于上方偏左。
- 口播字幕：底部居中，通常单行，白色粗体 + 黑色描边；收益、溢价、满房等重点词使用金黄色。
- 节点标题：五一、端午、暑期、中秋、国庆使用金黄色装饰字和深色描边。
- 关键词卡片：蓝底白字或暖黄勾选词，保持原帧时间和位置。
- 由于字幕与图形已烘焙在原视频中，最高保真策略不是 OCR 后重排，而是从原帧提取文字/卡片像素并在人物层之后回填。
- 顶部主标题大致位于 `x=95..495, y=120..325`；底部字幕安全区约为 `x=70..506, y=805..895`，基线约 `y=846`。
- 逐句字幕已按 0.5 秒采样提取并整理为 50 个事件，见 v2 JSON 的 `graphics.subtitle.events`。

## 音频与音效

- 0.00-67.68 秒为连续口播、背景音乐和转场音效的混合音轨。
- 原口播为女声，基频中位约 `216Hz`，10–90 分位约 `152–340Hz`，语速约 `303 个汉字/分钟`。
- 67.857-69.983 秒存在三组约 0.53、0.53、0.87 秒的 CTA 间歇，对应点赞/关注/评论动画节奏。
- 内部 1:1 技术验证可保留原混合音轨；正式原创发布改用黄丽已授权女声，并强制对齐原片约 303 字/分钟的字幕节奏。原 BGM/音效仅在确认授权后保留，否则换成节奏一致的自有音轨。
- 最终封装同步误差上限为 40ms，即 25fps 下最多 1 帧。

## 可执行生产链路

1. 从原片人物主画面重建无人物 clean plate。
2. 锁定最新企业身份资产 `asset://asset-20260711223536-zgjjn`，禁止回退素材 #10/#25/#31。
3. 按人物连续区间切成 5–10 秒参考片，将最新企业资产作为身份参考、原区间作为动作/构图参考提交 Seedance；不提交全屏 B-roll 和 CTA。
4. 对生成结果做人像抠图或背景一致性校正，合成到 L0。
5. 按 JSON 帧号回填原全屏 B-roll、顶部叠层、标题、字幕和卡片。
6. 复用原音轨并按 25fps 输出 1766 帧。
7. 做自动比对：切点误差 0 帧、背景相似度 >= 0.97、身份一致性 >= 0.90、文字像素差 <= 1%。

## 当前可行性判断

- 字幕、音效、B-roll、内容和结构 1:1：可实现，因为直接复用原帧/原音轨。
- 动作、表情和口型高一致：Seedance 多参考链路具备技术条件，必须先用最新企业资产做 5 秒技术样片验证。
- 黄丽整个人物与原背景同时像素级一致：不能依赖旧合成头像；应使用最新企业资产生成干净人物层，再与固定 clean plate 合成。
- 直接用 Seedance 生成整条：不可实现 1:1，已排除。

素材当前标记为 `reference_only`。内部技术验证可以继续；正式对外发布前，应确认原文案、B-roll、字幕设计、声音和音乐的使用授权。
