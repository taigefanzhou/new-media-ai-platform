# 参考视频深度拆解

把链接解析或上传得到的视频转成有证据、可复核、可用于原创生产的结构化蓝图。

## 工作流

1. 获取视频元数据、完整口播、句级时间段、镜头边界和关键帧。
2. 第一轮全片分析脚本结构、目标观众、镜头角色、字幕、声音和剪辑节奏。
3. 第二轮只复查钩子、转折、证据、高潮和行动引导；每条结论绑定时间范围。
4. 同时检查口播、视觉、画面文字、人物动作、音乐和音效；无法确认时明确标记。
5. 将复核结果写入 `reference_blueprint.deep_review`，并回填对应 `timeline` 片段。
6. 输出原创复用方案和可执行剪辑计划，交给脚本、数字人、素材匹配和视频生成模块。

## 必须输出

- 完整口播与时间段。
- 脚本钩子、叙事阶段、观点推进、证据和 CTA。
- 逐镜头时间、景别、机位、运镜、人物动作、字幕、声音和剪辑手法。
- 关键时刻的音视频证据、留存作用强度、薄弱点、优化建议和置信度。
- 第一轮结论与视频证据的矛盾及修正。
- 原创开场结构、叙事步骤、镜头节奏、字幕规则、声音规则和必须替换的元素。

## 判断边界

- `retention_score` 只表示内容结构的留存作用强度，不是真实播放或完播数据。
- 没有清晰证据时写入 `missing_evidence`，不得臆测。
- 结构、节奏和镜头方法可以学习；原文、人物、商家、音乐、标识和原画面必须替换。
- 第二轮失败时保留第一轮有效结果，不把整项分析判失败。
- 同一个火山视频文件只上传一次；3 分钟内默认两轮，长视频仅在第一轮质量不足时触发复核。

## 验收

- 关键结论能够追溯到具体时间段和口播或画面证据。
- 钩子、转折、证据、高潮和 CTA 至少被检查一次。
- 拆解结果能直接生成原创脚本、分镜和剪辑计划。
- 不把模型推测表述为真实平台数据。
- 不包含不可授权复刻的参考人物、台词、商家和画面。
